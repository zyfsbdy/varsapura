# Varsapura 交流群信息整合站
> 仅限 Varsapura 交流群内部成员使用的专属资讯与资源整合平台。

## 项目介绍
本项目是为 Varsapura 交流群搭建的**纯前端本地存储信息站**，无需后端服务器，部署于 GitHub Pages。
全站采用**邀请制 + 管理员审核**机制，确保信息安全与内部私密性。

## 核心功能
1. **严格的权限控制**
   - 游客：仅可浏览首页，无法点击任何功能按钮。
   - 注册：必须使用**管理员生成的邀请码**。
   - 审核：注册后需等待管理员审核通过方可登录。
   - 免登：登录状态永久保存，同一设备无需重复登录。

2. **资讯管理系统**
   - 支持资讯发布、编辑、删除。
   - 完整的资讯详情页，包含**浏览量统计**与**点赞功能**（防重复点赞）。
   - 支持按**时间、浏览量、点赞数**多维度排序与关键词搜索。

3. **管理员后台**
   - 生成唯一邀请码（一码一用）。
   - 人工审核新注册用户。

## 站点地图（文件结构）
| 文件名 | 功能描述 | 访问权限 |
| :--- | :--- | :--- |
| `index.html` | 首页（导航总入口） | 游客可看，功能需登录 |
| `login.html` | 账号登录页 | 公开 |
| `register.html` | 邀请制注册页 | 公开 |
| `admin.html` | 管理员后台（生成邀请码/审核） | 仅管理员 |
| `news.html` | 最新资讯列表页 | 已登录用户 |
| `edit-news.html` | 资讯发布/编辑页 | 已登录用户 |
| `news-detail.html` | 资讯详情页（点赞/浏览） | 已登录用户 |
| `invite-code.js` | 核心权限与数据存储脚本 | 公共依赖 |

## 管理员使用指南
1. **初始化**：首次进入 `admin.html`，生成邀请码并分发给群成员。
2. **用户审核**：用户注册后，在 `admin.html` 中找到对应账号，点击「审核通过」。
3. **资讯发布**：登录后通过 `index.html` 或 `news.html` 进入编辑页发布内容。

## 技术说明
- **存储方式**：使用 `localStorage` 存储用户、邀请码、资讯数据（数据存在用户本地浏览器中）。
- **部署方式**：GitHub Pages 静态站点部署。
- **兼容性**：支持现代浏览器（Chrome、Edge、Safari 等）。
